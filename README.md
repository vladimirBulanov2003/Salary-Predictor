# Salary Prediction from Job Postings

Проект по предсказанию зарплаты на основе описания вакансий с использованием глубокого обучения.

## Описание

Этот проект решает задачу регрессии - предсказание зарплаты по текстовым и категориальным признакам вакансии. Модель использует CNN-энкодеры для обработки текстовых данных (название и описание вакансии) и линейные слои для категориальных признаков. Файлик под названием Train_rev1.csv можно найти во 2 лабе (в семинаре) курса nlp от яндекса.  

## Архитектура модели

### Компоненты:

1. **Текстовые энкодеры (CNN-based)**:
   - **Title Encoder**:  сверточная сеть с окном размера 5
   - **Description Encoder**:  сверточная сеть с окнами размеров [4, 5, 6] и attention pooling
   - Используются предобученные GloVe эмбеддинги (50d)

2. **Категориальный энкодер**: линейный слой для обработки категориальных признаков

3. **Объединение и предсказание**: конкатенация всех признаков → dropout → линейный слой → log1p трансформация

### Используемые признаки:

- **Текстовые**: `Title`, `FullDescription`
- **Категориальные**: `Category`, `Company`, `LocationNormalized`, `ContractType`, `ContractTime`
- **Целевая переменная**: `SalaryNormalized` (логарифмируется как `Log1pSalary`)

## Технологии

- **PyTorch** - фреймворк для глубокого обучения
- **PyTorch Lightning** - высокоуровневая обертка для упрощения обучения
- **NLTK** - обработка и токенизация текста
- **scikit-learn** - предобработка данных (DictVectorizer, train_test_split)
- **GloVe embeddings** - предобученные векторные представления слов

## Особенности реализации

- Обработка текста: токенизация через NLTK WordPunctTokenizer, фильтрация редких токенов (частота < 8)
- Обработка категорий: DictVectorizer для кодирования, ограничение компаний до топ-1000
- Обучение: Adam optimizer (lr=1e-4), MSE loss, early stopping (patience=2)
- Логирование: TensorBoard для отслеживания метрик
- Checkpointing: автоматическое сохранение лучших моделей
